# Overview

> Component library for layout, emphasis, API documentation, and navigation.

Mintlify provides built-in MDX components for your documentation pages. Use these components to structure content, draw attention to important information, document APIs, and guide navigation.

 [TabsOrganize related content into switchable tabbed views.](https://mintlify.com/docs/components/tabs) [Code groupsDisplay code examples in multiple languages.](https://mintlify.com/docs/components/code-groups) [StepsDisplay sequential instructions in a numbered format.](https://mintlify.com/docs/components/steps) [ColumnsArrange content side by side in responsive layouts.](https://mintlify.com/docs/components/columns) [PanelCreate sidebar panels for supplementary content.](https://mintlify.com/docs/components/panel)

# Web editor overview

> Edit and publish documentation in your browser.

![Decorative image in light mode](https://mintcdn.com/mintlify/GiucHIlvP3i5L17o/images/editor/editor-light.png?fit=max&auto=format&n=GiucHIlvP3i5L17o&q=85&s=0579093c2743bb8f55c4f81bece9e902) ![Decorative image in dark mode](https://mintcdn.com/mintlify/GiucHIlvP3i5L17o/images/editor/editor-dark.png?fit=max&auto=format&n=GiucHIlvP3i5L17o&q=85&s=09938d1ca65739f048de43183a5bbaab) Create, edit, and publish documentation directly in your browser with the web editor. View and share previews of your changes; manage your site’s navigation structure with drag-and-drop components; and publish updates using your team’s preferred workflow.

## ​Key features

- **Visual and Markdown editing**: Switch between visual and Markdown editing modes as you work.
- **Live preview**: See real-time changes without creating a preview deployment.
- **Shareable preview links**: Create preview links for your team from the editor.
- **Drag-and-drop navigation**: Organize your site structure visually without editing configuration files.
- **Built-in media manager**: Upload and organize images, videos, and other assets.
- **Git synchronization**: All changes sync automatically with your documentation repository.
- **Branch workflows**: Work on feature branches and create pull requests for review.
- **Team collaboration**: Multiple team members can work simultaneously with preview deployments.

# Guides

> Learn how to create effective documentation with best practices and workflows.

The guides in this section contain advice for creating documentation that helps your users succeed. Whether you’re starting from scratch or improving existing docs, these guides help you plan, write, and maintain your documentation. Above all else, consider your users’ needs and goals. Document what helps them use your product and get back to their tasks.

## ​Topics

### ​AI tools

 Use AI-powered tools to accelerate your documentation workflow.

- [Automate documentation updates](https://mintlify.com/docs/guides/automate-agent): Tutorial that teaches how to set up automated documentation updates using the agent.
- [Build an in-app assistant](https://mintlify.com/docs/guides/assistant-embed): Tutorial that teaches how to add an AI chat bot in a React app to help users find answers from your documentation.
- [Claude Code](https://mintlify.com/docs/guides/claude-code), [Cursor](https://mintlify.com/docs/guides/cursor), [Windsurf](https://mintlify.com/docs/guides/windsurf): Guides for using AI coding assistants to maintain documentation.
- [GEO](https://mintlify.com/docs/guides/geo): Best practices for optimizing your documentation for AI search and answer engines.

### ​API documentation

 Create API documentation that lets users test and learn how to use your API, and is easy to maintain.

- [Migrate from MDX to OAS](https://mintlify.com/docs/guides/migrating-from-mdx): Move your API documentation from individual MDX pages to autogenerated pages from an OpenAPI specification.

### ​Best practices

 Make your documentation best in class.

- [Accessibility](https://mintlify.com/docs/guides/accessibility): Make your docs accessible to as many users as possible.
- [Content types](https://mintlify.com/docs/guides/content-types): Choose the right format for tutorials, how-tos, references, and explanations.
- [Content templates](https://mintlify.com/docs/guides/content-templates): Copy and modify templates for each content type.
- [Improve your docs](https://mintlify.com/docs/guides/improving-docs): Use data and feedback to continuously improve your documentation.
- [Internationalization](https://mintlify.com/docs/guides/internationalization): Set up multi-language documentation to reach global audiences.
- [Linking](https://mintlify.com/docs/guides/linking): Create internal links, reference API endpoints, and maintain link integrity across your documentation.
- [Maintenance](https://mintlify.com/docs/guides/maintenance): Keep your documentation accurate and up-to-date.
- [Media](https://mintlify.com/docs/guides/media): Add images, videos, and other media effectively while managing maintenance burden.
- [Organize navigation](https://mintlify.com/docs/guides/navigation): Design information architecture that works for your users.
- [SEO](https://mintlify.com/docs/guides/seo): Optimize your docs for search engines.
- [Style and tone](https://mintlify.com/docs/guides/style-and-tone): Write with a consistent voice.
- [Understand your audience](https://mintlify.com/docs/guides/understand-your-audience): Research and serve your users’ needs and goals.

### ​Git workflows

 Understand Git to get the most out of Mintlify and a docs-as-code approach to documentation.

- [Git concepts](https://mintlify.com/docs/guides/git-concepts): Learn the fundamentals of Git for documentation.
- [Branches](https://mintlify.com/docs/guides/branches): Use branches to manage documentation changes.

### ​Use cases

 Learn how Mintlify supports different use cases and find a solution that works for you.

- [Create developer documentation](https://mintlify.com/docs/guides/developer-documentation): Help your users get the most from your product or API with developer documentation that answers questions and gets people back to their main tasks.
- [Create a knowledge base](https://mintlify.com/docs/guides/knowledge-base): Set up an internal knowledge base to consolidate information for your team, improve search, and reduce maintenance burden.
- [Create a support center](https://mintlify.com/docs/guides/support-center): Build a self-service support center that helps customers find answers, reduces ticket volume, and improves customer satisfaction.
